# Support Lead Checklist (pre-release & post-release)

Pre-release (support readiness)
- [ ] On-call rotation and contacts documented
- [ ] Runbooks for expected failure modes created/updated
- [ ] Monitoring & alerts configured for new features
- [ ] Support triage playbook communicated to stakeholders
- [ ] Post-release verification tasks assigned

Post-release (incident & feedback handling)
- [ ] Incident triage completed and ticket created
- [ ] Severity & customer-impact assessment recorded
- [ ] Short-term mitigation action assigned
- [ ] Root cause analysis scheduled (if needed)
- [ ] Follow-up action items added to backlog with owners

# Release Checklist (supplemental)

- [ ] Acceptance criteria verified and tests passing
- [ ] Rollback / mitigation plan documented
- [ ] Release notes drafted and reviewed
- [ ] Support readiness checks (see Support Lead)
- [ ] Stakeholder notification scheduled
- [ ] Post-deploy verification steps defined and assigned
